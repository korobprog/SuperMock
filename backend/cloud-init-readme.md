# Инструкция по использованию Cloud-init для настройки сервера

В этом документе описано, как использовать Cloud-init для автоматической настройки сервера под Docker-сервисы проекта SuperMook.

## Что такое Cloud-init

Cloud-init — это инструмент для настройки облачного сервера, который позволяет передать пользовательские параметры (user-data) при запуске виртуальной машины и тем самым ускорить и автоматизировать процесс настройки серверов.

С помощью cloud-init вы можете настроить конфигурацию сервера, установить нужное ПО, создать нужные директории и пользователей и назначить им права — и многое другое.

## Форматы Cloud-init

В этом проекте предоставлены два формата Cloud-init:

1. **YAML-формат** (`cloud-init.yaml`) - стандартный формат Cloud-init, который поддерживается большинством облачных провайдеров.
2. **Shell-скрипт** (`cloud-init.sh`) - альтернативный формат, который может быть использован, если YAML-формат не работает корректно.

## Как использовать Cloud-init в Timeweb Cloud

### Шаг 1: Создание нового сервера

1. Войдите в панель управления Timeweb Cloud
2. Выберите "Создать" > "Виртуальный сервер"
3. Выберите нужную конфигурацию сервера (ОС, CPU, RAM, диск)
4. На шаге настройки найдите раздел "Cloud-init"

### Шаг 2: Загрузка скрипта Cloud-init

У вас есть два варианта:

#### Вариант 1: Использование YAML-формата

1. Откройте файл `cloud-init.yaml` в текстовом редакторе
2. Скопируйте содержимое файла
3. Вставьте в поле Cloud-init в панели управления Timeweb Cloud
4. Убедитесь, что выбран формат "#cloud-config"

#### Вариант 2: Использование Shell-скрипта

1. Откройте файл `cloud-init.sh` в текстовом редакторе
2. Скопируйте содержимое файла
3. Вставьте в поле Cloud-init в панели управления Timeweb Cloud
4. Убедитесь, что выбран формат "#!/bin/sh"

### Шаг 3: Завершение создания сервера

1. Нажмите "Создать сервер"
2. Дождитесь создания сервера и выполнения скрипта Cloud-init

## Что делает скрипт Cloud-init

Скрипт Cloud-init выполняет следующие действия:

1. Обновляет пакеты системы
2. Устанавливает Docker и Docker Compose
3. Создает директорию `/app` для приложения
4. Создает файлы `docker-compose.yml` и `.env` с настройками для вашего проекта
5. Запускает Docker-контейнеры с бэкендом, MongoDB и Redis
6. Настраивает автозапуск Docker при загрузке сервера
7. Настраивает ежедневное обновление Docker-образов
8. Создает пользователя `admin` с правами sudo
9. Добавляет логи для отладки в файл `/var/log/cloud-init-custom.log`

## Проверка работы

После завершения настройки сервера вы можете проверить работу приложения:

1. Откройте в браузере `http://217.198.6.238:80/api` для проверки API
2. Используйте `http://217.198.6.238:80` для WebSocket-соединения

## Логи и отладка

Если возникли проблемы, вы можете проверить логи Cloud-init:

```bash
cat /var/log/cloud-init-output.log
cat /var/log/cloud-init-custom.log
```

Для проверки запущенных контейнеров:

```bash
docker ps
```

Для проверки открытых портов:

```bash
netstat -tulpn | grep LISTEN
```

## Настройка портов

Если некоторые порты закрыты на уровне провайдера, вы можете открыть их в панели управления Timeweb Cloud:

1. Выберите ваш сервер в панели управления
2. Перейдите в раздел "Брандмауэр" или "Группы безопасности"
3. Добавьте правила для открытия портов:
   - 8080 (HTTP для бэкенда)
   - 80 (API и WebSocket)
   - 27017 (MongoDB)
   - 6379 (Redis)

## Обновление приложения

Для обновления приложения выполните:

```bash
cd /app
docker-compose pull
docker-compose up -d
docker image prune -af
```

Или дождитесь автоматического обновления, которое происходит каждый день в 3:00.
