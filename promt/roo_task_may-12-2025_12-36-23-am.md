# Отладка и решение проблем с SuperMook

## Описание проблемы

При запуске приложения SuperMook на сервере возникли следующие проблемы:

1. **Проблема с MongoDB URI** - в контейнере используется имя сервиса "mongo", а не IP-адрес, что может вызывать ошибки аутентификации
2. **Отсутствие файлов бэкенда на сервере** - в скрипте restart.sh копируется только server.ts, но не копируются другие необходимые файлы
3. **Проблема с отсутствием файлов фронтенда** - бэкенд не может найти файлы в директории /react-frontend/dist/
4. **Проблема с портами и маршрутизацией** - несоответствие портов в разных конфигурационных файлах
5. **Проблема с настройками CORS** - возможные ограничения при обращении к API с фронтенда

## Анализ возможных источников проблемы

### 1. Проблема с MongoDB URI

- В контейнере Docker используется имя сервиса "mongo" вместо IP-адреса
- Это может вызывать проблемы с разрешением имен в сети Docker
- Возможны проблемы с аутентификацией при подключении к MongoDB

### 2. Отсутствие файлов бэкенда на сервере

- Скрипт restart.sh копирует только файл server.ts
- Отсутствуют другие необходимые файлы (middleware, routes, config)
- Это приводит к ошибкам при импорте модулей в server.ts

### 3. Проблема с отсутствием файлов фронтенда

- Бэкенд не может найти статические файлы фронтенда
- Директория /react-frontend/dist/ может отсутствовать или иметь неправильные права доступа
- Возможно, фронтенд не был собран перед деплоем

### 4. Проблема с портами и маршрутизацией

- Несоответствие портов в разных конфигурационных файлах
- Возможные конфликты портов на сервере
- Неправильная настройка проксирования в nginx

### 5. Проблема с настройками CORS

- Неправильная настройка CORS в middleware
- Ограничения при обращении к API с фронтенда
- Отсутствие необходимых заголовков в ответах API

## Наиболее вероятные источники проблемы

После анализа всех возможных источников проблемы, наиболее вероятными являются:

1. **Отсутствие файлов бэкенда на сервере** - скрипт restart.sh копирует только server.ts, но не копирует другие необходимые файлы, что приводит к ошибкам при импорте модулей
2. **Проблема с MongoDB URI** - неправильная настройка подключения к MongoDB в контейнере Docker

## Решение проблемы

На основе анализа файлов и подтверждения, основная проблема заключается в отсутствии необходимых файлов бэкенда на сервере. Скрипт restart.sh копировал только файл server.ts, но не копировал другие необходимые файлы, такие как middleware/cors.ts, config/app.ts и т.д.

### Внесенные изменения:

1. **Добавлено расширенное логирование в server.ts:**

   - Добавлена проверка наличия всех необходимых файлов при запуске
   - Добавлено подробное логирование процесса подключения к MongoDB
   - Исправлена ошибка TypeScript с параметром err в функции catch

2. **Изменен скрипт restart.sh:**
   - Теперь скрипт копирует всю директорию backend/src/ на сервер, а не только один файл
   - Добавлены команды для проверки структуры директорий на сервере
   - Добавлено дополнительное логирование для отладки

### Как проверить решение:

1. Запустите обновленный скрипт restart.sh для копирования всех файлов на сервер:

```bash
cd backend
chmod +x restart.sh
./restart.sh
```

2. После перезапуска контейнеров проверьте логи MongoDB:

```bash
ssh root@217.198.6.238 "docker logs supermock-mongo-1"
```

3. Проверьте логи бэкенда для подтверждения, что все файлы найдены:

```bash
ssh root@217.198.6.238 "docker logs supermock-backend-1"
```

Теперь бэкенд должен успешно запуститься со всеми необходимыми файлами и корректно подключиться к MongoDB.
